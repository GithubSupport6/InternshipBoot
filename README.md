Приложение должно реализовать следующий сценарий работы:

1 Пользователь запускает приложение из командной строки mvn jetty:run

2 Пользователь открывает браузер и переходит по ссылке http://localhost:8080

3 Открывается страница, на которой отображается следующее

3.1 Текущая дата компьютера

3.2 Название компьютера

3.3 Свободная память Java

3.4 Всего памяти Java

3.5 Поля для ввода новой задачи

Номер задачи (число):____________

Описание задачи (мнгострочный текст):_________

Срок выполнения задачи (дата):____

[Добавить задачу]

3.6 Список задач пользователя в виде таблицы
Номер задачи	Описание задачи	Срок выполнения	Действие
1	Выполнить тестовое задание	03.07.2019	[Изменить][Удалить]

 3.7 Пользователь вводит номер задачи, описание, срок выполнения и нажимает на Добавить задачу, задача без перезагрузки страницы появлется в списке задач и записывается на сервере в h2database

3.8 Пользователь нажимает кнопку Изменить у задачи, свойства задачи отображаются в полях над таблицей, скрывается кнопка [Добавить задачу], вместо неё появляются кнопки [Сохранить] [Отмена]

3.9 При нажатии на кнопку [Сохранить] данные из полей сохраняются в свойствах задачи на сервере и обновляются в списке задач, кнопки  [Сохранить] [Отмена] скрываются, поля очищаются, показывается кнопка [Добавить задачу]

3.10 При нажатии на кнопку [Отмена] кнопки  [Сохранить] [Отмена] скрываются, поля очищаются, показывается кнопка [Добавить задачу]

3.11 При обновлении страницы по F5 список задач загружается с сервера

3.12 При реализации использовать REST, на клиентской стороне использовать YUI 2.7

3.13 При вводе задач или изменении их проверять на корректность полей (маска ввода и проверка на пустые поля).
